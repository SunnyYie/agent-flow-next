# Skill: 源码搜索与学习

## Trigger — 何时使用

遇到实现问题，需要参考开源项目的实现方式时。不限特定项目，根据问题自主决定搜索目标。

## Required Reading — 前置阅读

- `.dev-workflow/Agent.md` 第2.5节 思维链准则

## Procedure — 执行步骤

1. **明确搜索目标**：需要找到什么类型的实现？用一句话描述问题。
2. **并行搜索**（4种工具同时启动）：
   - `gh search repos "{关键词}" --limit 5` — 搜索相关开源项目
   - `gh search code "{关键词}" --language python --limit 10` — 搜索代码片段
   - `WebSearch "{关键词} implementation best practice"` — 搜索技术文档和博客
   - `find-docs "{技术名}"` — 搜索权威技术文档
3. **筛选结果**：从搜索结果中选择最相关的1-2个项目，优先选择星标高、最近更新的项目。
4. **阅读源码**（实现级深度）：
   - 先看项目结构和README理解架构
   - 再看具体模块的实现代码
   - 关注：接口设计、错误处理、测试模式
5. **提炼与记录**：
   - 提炼可借鉴的设计模式（不直接复制代码）
   - 记录搜索关键词和路径，便于复现
   - 写入参考笔记或SOUL.md

## Rules — 规则约束

- 优先选择星标高、最近更新的项目
- 不直接复制代码，只借鉴设计模式
- 记录搜索关键词和路径，便于复现
- 参考笔记存入 `.dev-workflow/shared/references/`，按项目建文件
- 搜索策略导向：重点记录"怎么找到答案的"，而非仅记录答案本身
