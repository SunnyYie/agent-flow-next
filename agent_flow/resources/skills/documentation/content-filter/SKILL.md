# Skill: content-filter

## Trigger
当需要对文档内容进行过滤、清洗、脱敏或提取时触发

## Procedure
1. **查找过滤规范**（必须先做）：
   - 搜索 Skill/Wiki 中是否有该类型内容的过滤规则
   - WebSearch 搜索"{文档类型}内容过滤规范"或"{文档类型}敏感信息处理"
   - 如果找到规范 → 严格按规范执行
   - 如果没找到 → 列出过滤规则草案，请求用户确认
2. **识别过滤类型**：
   - **脱敏**：去除敏感信息（密钥、token、内部URL、账号密码）
   - **去冗余**：去除重复、矛盾、无关内容
   - **精炼**：提取核心信息，去除装饰性描述
   - **合规**：去除不符合目标格式规范的内容
3. **执行过滤**：
   - 第一遍：脱敏 — 标记并移除所有敏感信息
   - 第二遍：去冗余 — 去除重复和无关内容
   - 第三遍：精炼 — 提取核心，去除装饰性描述
   - 第四遍：合规 — 检查是否符合目标格式规范
4. **验证过滤结果**：
   - 检查是否有遗漏的敏感信息
   - 检查是否误删了关键业务数据
   - 检查过滤后的内容是否完整连贯

## 常见过滤规则

### 飞书/文档类
| 过滤项 | 规则 | 示例 |
|--------|------|------|
| image token | 移除 `<image token="xxx"/>` 标签 | `<image token="KPsbbS..."/>` → 删除 |
| sheet token | 移除 `<sheet token="xxx"/>` 标签 | `<sheet token="WZoos3..."/>` → 删除 |
| bitable token | 移除 `<bitable token="xxx"/>` 标签 | `<bitable token="VyvSbt..."/>` → 删除 |
| 内部链接 | 转换为描述性文字 | `https://maimai.cn/...` → 删除或标注为"原始链接" |
| 高亮标注 | 提取文本，去除格式 | `<text bgcolor="light-yellow">xxx</text>` → `xxx` |

### 代码/配置类
| 过滤项 | 规则 |
|--------|------|
| 密钥/Token | 替换为 `{REDACTED}` |
| 内部IP | 替换为 `{INTERNAL_IP}` |
| 用户邮箱 | 替换为 `{USER_EMAIL}` |
| 调试信息 | 整段删除 |

### 通用冗余内容
| 过滤项 | 规则 |
|--------|------|
| 编写目的/背景 | 需求规格说明书中删除 |
| 项目管理占位 | "待补充/待定"的内容删除或标注"原始文档未提供" |
| 重复描述 | 保留最完整的版本，删除其他重复 |
| 装饰性文字 | 去除无实质内容的修饰语 |

## Rules
- 必须先搜索过滤规范再执行，不能自行判断哪些内容"冗余"
- 脱敏优先于去冗余——先确保不泄露敏感信息
- 去冗余时保守操作——宁可保留也不要误删关键数据
- 过滤后必须验证完整性，确保没有误删
