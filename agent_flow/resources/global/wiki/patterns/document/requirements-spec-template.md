# 需求规格说明书标准模板

## 适用场景

将产品需求文档（PRD）、飞书Wiki等原始需求资料转换为结构化的需求规格说明书（SRS）时使用。

## 模板结构

需求规格说明书聚焦功能需求，采用**四段式结构**：

```
# 需求规格说明书：{产品/功能名称}

## 1 功能需求

### 1.1 核心流程
{整体业务流程的步骤概览，编号列表}

### 1.2 {功能模块1}
{详细功能规则、输入输出、业务逻辑}

### 1.3 {功能模块2}
...

### 1.N {功能模块N}
...

## 2 非功能需求

| 约束项 | 规则 |
|--------|------|
| 作用范围 | {一期/全量/特定场景} |
| 频率限制 | {调用/操作频率约束} |
| 审核规则 | {是否走平台审核} |
| 字数限制 | {输出长度约束及截取规则} |
| 其他约束 | {模型/性能/兼容性等} |

## 3 数据埋点

- 评论/内容打标规则（如待确认则标注"待确认"）
- 标签名字：{标签名}
- 标签id：{id或"待确认"}

| 页面 | 事件名称 | 事件id | 参数 | 参数类型 | 参数值 | 备注 |
|------|----------|--------|------|----------|--------|------|
| ... | ... | ... | ... | ... | ... | 新增/已有埋点 |

## 4 待确认事项

- {原文中标注"？？"或"待定"的项}
- {边界条件未明确的项}
- {技术实现细节待定的项}
```

## 内容过滤规则

### 必须保留
- 功能需求：触发逻辑、业务规则、输入输出、Prompt全文、输出模板
- 非功能需求：所有约束条件（频率、审核、字数、模型等）
- 数据埋点：事件id、参数名、参数值（与原文完全一致）
- 业务配置：uid、模型名、字数限制等精确数据

### 必须去除
- 引言/编写目的/项目背景 → 冗余元信息
- 项目管理占位符（提测时间、上线时间等空字段）
- 相关链接（设计稿、jira等空链接）
- 内部技术标识（`<image token>`, `<sheet token>`, `<bitable token>`）
- 内部URL链接

### 提取转换
- `<text bgcolor="light-yellow">xxx</text>` → 纯文本 `xxx`
- `<text color="red">xxx</text>` → 纯文本 `xxx`
- `<quote-container>xxx</quote-container>` → 纯文本 `xxx`
- 高亮中的"？？" → "待确认"

## 关键原则

1. **关键字段与原文完全一致**：输出模板、字数限制、Prompt、uid、事件id等不能意译
2. **四段式结构不增不减**：功能需求/非功能需求/数据埋点/待确认事项，不添加引言和项目管理
3. **待确认事项必须标注**：原文中任何不确定项都要在"待确认事项"中列出
4. **保守过滤**：宁可保留也不要误删关键业务数据

## 验收标准

| 维度 | 检查项 |
|------|--------|
| 完整性 | 功能需求/非功能需求/数据埋点/待确认事项全覆盖 |
| 准确性 | 关键字段（模板、字数、uid、事件id）与原文一致 |
| 结构 | 四段式，无冗余元信息 |
| 过滤 | 无内部技术标识，无内部URL，高亮文本已提取 |
| 待确认 | 原文不确定项均已标注 |
