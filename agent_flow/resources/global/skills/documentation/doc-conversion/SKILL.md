# Skill: doc-conversion

## Trigger
当需要将原始文档转换为特定格式的输出文档时触发（如：需求文档→规格说明书、会议纪要→行动项、调研报告→决策文档）

## Procedure
1. **查找目标格式规范**（必须先做）：
   - 搜索 Skill/Wiki 中是否有目标格式的模板或规范
   - WebSearch 搜索"{目标文档类型}标准模板"或"{目标文档类型}格式规范"
   - 如果找到规范 → 严格按规范组织
   - 如果没找到 → 询问用户期望的结构
2. **解析原始文档**：
   - 识别文档结构（章节、表格、列表）
   - 提取关键信息（业务逻辑、功能点、约束条件、数据定义）
   - 标记文档类型（需求文档、设计文档、会议纪要等）
3. **内容过滤**（必须先搜索过滤规范）：
   - 搜索 Skill/Wiki 中是否有该类型文档的内容过滤规则
   - 执行 [[content-filter|skills/documentation/content-filter/handler]] 技能进行过滤
   - 保留所有业务关键数据（配置值、阈值、ID等）
4. **格式转换**：
   - 按照找到的规范模板组织内容
   - 关键字段（数字、模板、限制条件）与原文完全一致，不能意译
   - 原文中"待补充/未提供"的内容标注为"原始文档未提供"
5. **自检**：对照规范逐项检查是否遗漏

## 常见目标格式参考

### 需求规格说明书
- 核心章节：功能需求、非功能需求、数据定义、待确认事项
- 不需要：引言/背景/编写目的/项目管理占位符
- 关键：保留所有业务规则、约束条件、字数限制等精确数据

### 技术方案文档
- 核心章节：方案概述、架构设计、接口定义、部署方案
- 关键：保留技术选型理由、性能指标、兼容性要求

### 会议纪要
- 核心章节：决策项、行动项（负责人+截止日期）、遗留问题
- 过滤：寒暄、重复讨论、与决策无关的细节

## Rules
- 必须先搜索格式规范再执行，不能自行臆断格式
- 必须先搜索内容过滤规则再过滤，不能自行判断哪些内容冗余
- 关键数据（数字、模板、限制）与原文保持完全一致
- 去除所有内部技术标识，保留业务配置信息
